Flexbox

Flex container_flex - это элемент, которому задано свойство display со значениями flex или inline-flex
Элементы внутри контейнера называются флекс элементами (flex Items)
Элемент может быть как один, так их может и быть множество. Размер элементов зависит от размера контента внутри
По умолчанию, элементы располагаются слева-направо и сверху-вниз

Основаная и дополнительная оси

Существует две оси: основная (Main Axis) и перпендикулярной ей дополнительная или поперечная (Cross Axis)
Все элементы флекс контейнера выстраиваются вдоль основной оси
Дополнительная ось служит для выравнивания элементов друг относительно друга
Таким образом, флекс бокс - это блок, элементы внутри которого размещаются по горизонтали и выравниваются по вертикали
Оси являются основой принципа флекс бокс, а благодаря изменению их направлений, осуществляется изменение направления самих элементов, которые выстраиваются по ним

flex-direction - ориентация главной оси

Свойство определяет ориентацию основной оси, позволяя напрявлять горизонтально или вертикально элементы внутри контейнера
Изменяется именно направление оси, а не порядок самих элементов (за это во флекс бокс отвечает свойство order)
Имеет значения:
- row (основная ось слева направо, значение по умолчанию): элементы располагаются горизонтально слева направо;
- row-reverse (основная ось справа налево): элементы располагаются горизонтально справа налево;
- column (основная ось сверху вниз): элементы располагаются вертикально сверху вниз;
- column-reverse (основная ось снизу вверх): элементы располагаются вертикально снизу вверх;

flex-wrap - перенос элементов

Свойство определяет, должны ли элементы flex-контейнера переноситься на новую строку или столбец при заполнении доступного пространства. Имеет значения:
- nowrap (значение по умолчанию): элементы внутри флекс контейнера не переносятся на новую строку, они остаются на одной строке и занимают всю её доступную ширину;
- wrap: указывает элементам flex-контейнера переноситься на новую строку, если они не помещаются в одну линию;
- wrap-reverse (разворачивает дополнительную поперечную ось на 180 градусов): элементы переносятся на новую строку или столбец в обратном порядке, последний элемент может стать первым на следующей строке, а первый элемент может стать последним на предыдущей строке;
- inherit: значение наследует свойство flex-wrap от родительского элемента. Это позволяет элементу flex-контейнера наследовать поведение по переносу элементов на новую строку или столбец

Для лучшей демонстрации, элементам задана ширина 300px

nowrap

Ввиду недоступности переноса, элементы не реализуют заданную им ширину, а делят между собой доступное пространство

Гибкий элемент карточек на всю ширину

Флекс контейнеры и флекс элементы воспринимают свойство margin и padding
Однако, наличие отступов у элементов, приводит к нехватке достаточного места для всех элементов в строке, образуя пустое пространство

Наличие у элементов margin и padding

Изначальный вид трехколоночного лейаута, но с недостаточным количеством свободного места для размещения элементов на всей ширине
Наличие margin в 5px и padding в 8px с обеих сторон у каждого элемента, не позволяет элементам строки занимать всю ширину

Применение для элементов трехколоночного лейаута функции calc(), с учетом всех отступов каждого элемента
Например, общий размер каждого элемента включает: 33.33% от ширины контейнера + 5px по сторонам (margin) + 8px по сторонам (padding), что можно учесть в функции для задания гибкой ширины каждого элемента: width: calc(33.33% - 10px - 16px);

flex-flow - комбинация flex-direction и flex-wrap

Свойство является сокращенной формой для одновременной установки свойств: flex-direction, для расположения главной оси (горизонтальное или вертикальное расположение элементов), а также flex-wrap, для определения переноса флекс элементов на новую строку (если на текущей не будет достаточно места)
Синтаксис
flex-flow: <flex-direction> <flex-wrap>

Одновременное задание контейнеру значений столбца и переноса элементов

.flex_flex-flow1 {
    flex-flow: column wrap;
    height: 150px;
}
.flex_flex-flow1 .flex-item {
    width: 150px;
}

align-self

Свойство аналогичное align-items, но переопределяющее его значения

Переопределение свойств элементов

- Переопределение первого элемента на stretch. Повдение по умолчанию растягивает элемент по поперечной оси

- Переопределение свойства четвертого элемента на center, перенесет элемент в центр оси
- Переопределение свойства шестого элемента на baseline, перенесет элемент к базовой линии основной оси, что при однородности элементов приведет к виду flex-start
- Переопределение свойства седьмого элемента на baseline с изменением шрифта элемента, для наглядного смещения базовой линии

flex-grow

Определяет пропорцию распределения доступного свободного пространства для конкретного flex-элемента, относительно других (с установленным значением)
Применяется к флекс элементам

flex-shrink

Определяет пропорцию сжатия конкретного flex-элемента относительно других, при распределении отрицательного свободного пространства.
Применяется к флекс элементам
Стандартное значение: 1

Свойство принимает положительное целое или дробное числовое значение, которое определяет, какую часть недостающего пространства должен занять элемент

Установленное значение

Для всех элементов установлено значение ширины в 200px
Для второго элемента установлено значение flex-shrink: 2
Для третьего элемента установлено значение flex-shrink: 3

Соответственно, при уменьшении размеров контейнера, когда элементам перестанет хватать размера, элементы без свойства flex-shrink станут сжиматься (делиться свободным пространством) в обычном режиме
Тогда как второй и третий элементы сжимаются (делятся свободным пространством) больше, при этом третий "отдаст" больше второго

flex-basis

Определяет начальный основной размер flex-элемента до распределения свободного пространства в соответствии с коэффициентами гибкости (перед тем как к нему будут применены свойства flex-grow и flex-shrink)
Применяется к флекс элементам
Стандартное значение: auto

Разница заключается в том, что значение flex-basis имеет больший приоритет по сравнению со значениями width и height, и является базисом для расчета flex-grow и flex-shrink.

Именно basis рекомендуется применять для установки размеров элементам, одновременно определяя ширину или высоту элемента вдоль главной оси
- При изначальном горизонтальном положении оси (flex-direction: row) - устанавливает ширину;
- При вертикальном положении оси (flex-direction: column) - устанавливает высоту. Подобного повдения нельзя добиться, устанавливая размеры элементов другим свойством, например width

flex-basis: 40px

Элементам установлена ширина в 40px. Они могут сжиматься со стандартным поведением, но имеют максимальную ширину в 40px

.flex_flex-basis40 .flex-item {
    flex-basis: 40px;
}

flex-direction: row
flex-basis определяет размер элементов по основной оси (горизонтали), задавая изначальную ширину элементов

Гибкий элемент карточек

flex-direction: column;
flex-wrap: wrap;
height (контейнера): 600px;
flex (элементов): 1 0 100px;

Базовый размер установлен в 100px, что при flex-direction: column станет задавать базовую высоту элементов
Вместе со значением высоты контейнера, заданной в 600px и разрешением на перенос элементов на новую строку, это приводит к созданию двухколоночного лейаута
При уменьшении контейнера, элементы, которым будет недостаточно пространства для базового размера, станут переноситься на следующий ряд
При увеличении контейнера, в конечном счете, элементы сформируют один столбец из элементов, как это происходит для подобных контейнеров, при незаданной высоте, когда контейнер подстраивается под размер содержимого
А благодаря применению flex-grow (первому значению в flex:...), элементам удается растянуться на всё пространство контейнера

flex

Является сокращением от flex-grow, flex-shrink и flex-basis
Элемент будет растягиваться на всю доступную ширину (или высоту в зависимости от направления оси) в соответствии с размерами контейнера и другими дочерними элементами.
Если другие дочерние элементы также имеют свойство flex, размеры элементов будут распределены пропорционально их значениям

flex: initial; - элемент сохраняет свой исходный размер flex: auto; - элемент растягивается или сжимается в зависимости от размера контейнера flex: none; - элемент не растягивается и сохраняет свой исходный размер по ширине

Растягивание первого элемента и задание фиксированных размеров для остальных c возможностью их сжатия

.el_special_flex_0_1_200px .flex-item {
    flex: 0 1 200px;
}
.el_special_flex_0_1_200px .item-1 {
    flex: 1;
}

Для всех элементов установлена изначальная шиирина в 200px, с возможностью сжиматься, но без возможности роста. Достигая своего базового размера, элементы перестают расти
В тоже время, первому элементу задана возможность занимать всё доступное пространство контейнера

order

Определяет порядок вывода флексов внутри флекс-контейнера
Свойство принимает целое положительное или отрицательное числовое значение, а также 0, что определяет, в каком порядке элемент должен выводиться
Элементы располагаются согласно значениям свойства order от меньшего к большему. При равных значениях order, элементы выводятся в том порядке, в каком они указаны в разметке

Отображение третьего элемента первым

    order: -3;
}

Пример упорядочивания элементов, при viewidth менее 960px

Трехколоночный макет, который изменяет свое положение на столбец, при вьюпорте менее 960px
При этом, для того, чтобы два боковых меню не шли в самом начале, изменяется порядок отображения блоков
Так, при вьюпорте менее 960px, макет меняет свое положение на столбец из трех блоков, а также изменяет порядок отображения блоков, где сначала отображается основная часть, а после неё два блока меню 

.example_order_for_menu .flex-item {
    font-size: 2rem;
    flex-grow: 1;
}
@media screen and (max-width: 960px) {
    .example_order_for_menu {
        flex-direction: column;
    }
    .example_order_for_menu .item-2 {
        order: -1;
    }
}

Трехколоночный лейаут страницы

Header и Footer растягиваются на всю ширину экрана. Блок main масштабируется для пропорционального отображения двух aside и секции с основным контентом

Блок мейн представляет собой вложенный флекс контейнер, который определяет собственные правила для позиционирования элементов
Где свободное пространство делится между элементами aside и section в пропорциях 1:3, чтобы основная часть была шире остальных

Оптимизация включает отображение одноколоночного лейаута, при viewport менее 960px
Блок main перестроится для отображения контента с flex-direction: row на flex-direction: column

Полезной особенностью является фиксирование положения header и footer на странице, даже если будет удален контент в aside и section. Сохраненные родители станут иметь форму, что создаст вид фиксирования мета блоков
