Flex

- Flex Container - это элемент, которому задано свойство display со значениями flex или inline-flex
- Элементы внутри Flex Conteiner называются флекс элементами (flex Items)
Элемент может быть как один, так их может и быть множество. Размер элементов зависит от размера контента внутри

Расположение элементов

По умолчанию, элементы располагаются слева-направо и сверху-вниз

Основаная и дополнительная оси

Существует две оси: основная (Main Axis) и перпендикулярной ей дополнительная или поперечная (Cross Axis)
Все элементы флекс контейнера выстраиваются вдоль основной оси
Дополнительная ось служит для выравнивания элементов друг относительно друга
Таким образом, флекс бокс - это блок, элементы внутри которого размещаются по горизонтали и выравниваются по вертикали
Оси являются основой принципа флекс бокс, а благодаря изменению их направлений, осуществляется изменение направления самих элементов, которые выстраиваются по ним

flex-direction - ориентация главной оси

Свойство определяет ориентацию основной оси, позволяя напрявлять горизонтально или вертикально элементы внутри контейнера
Изменяется именно направление оси, а не порядок самих элементов (за это во флекс бокс отвечает свойство order)
Имеет значения:
- row (основная ось слева направо, значение по умолчанию): элементы располагаются горизонтально слева направо;
- row-reverse (основная ось справа налево): элементы располагаются горизонтально справа налево;
- column (основная ось сверху вниз): элементы располагаются вертикально сверху вниз;
- column-reverse (основная ось снизу вверх): элементы располагаются вертикально снизу вверх;

flex-wrap - перенос элементов

Свойство определяет, должны ли элементы flex-контейнера переноситься на новую строку или столбец при заполнении доступного пространства. Имеет значения:
- nowrap (значение по умолчанию): элементы внутри флекс контейнера не переносятся на новую строку, они остаются на одной строке и занимают всю её доступную ширину;
- wrap: указывает элементам flex-контейнера переноситься на новую строку, если они не помещаются в одну линию;
- wrap-reverse (разворачивает дополнительную поперечную ось на 180 градусов): элементы переносятся на новую строку или столбец в обратном порядке, последний элемент может стать первым на следующей строке, а первый элемент может стать последним на предыдущей строке;
- inherit: значение наследует свойство flex-wrap от родительского элемента. Это позволяет элементу flex-контейнера наследовать поведение по переносу элементов на новую строку или столбец

flex-flow - комбинация flex-direction и flex-wrap

Свойство является сокращенной формой для одновременной установки свойств: flex-direction, для расположения главной оси (горизонтальное или вертикальное расположение элементов), а также flex-wrap, для определения переноса флекс элементов на новую строку (если на текущей не будет достаточно места)

Формат значений для свойства упорядочен. flex-flow: <flex-direction> <flex-wrap>

Размещение элементов вертикально, с разрешением для нимх переноса, через одно свойство - flex-flow. Значением для свойства станут: column и wrap
Для самого контейнера фиксируется высота, например 150px, чтобы отобразить флекс элементы в колонках, иначе контейнер будет растянут под контент внутри
А исходя из высоты контейнера, элементы подстраиваются относительно главной оси и следуют сверху вниз и слева направо, таким образом формируя столбцы контента из доступного пространства контейнера

Отступы между флекс элементами

Флекс контейнеры и флекс элементы воспринимают свойство margin и padding
Так, с установленным свойством margin, элементы из примера выше, станут иметь дополнительные отступы вокруг себя. Итого, общий размер каждого элемента = 33.33% от ширины + 5px со всех сторон
Что приводит к нехватке места для всех элементов в строке и образуется пустое пространство

Gap

Определение расстояния между флекс элементами
Применяется как более продвинутая система настройки отступов между флекс элементами, например, вместо свойства margin
Так, например, вместо margin 5px, может быть применен gap для задания расстояния между элементами, благодаря свободному пространству от блоков до flex (а не inline-flex) контейнера
Инструмент не идеален. Для flex контейнера (не inline-flex) потребуется вычислять доступное расстояние вручную

Выравнивание элементов

Свойства для флекс контейнера (Container Alignment):

- justify-content - выравнивание вдоль основной оси
- align-items - выравнивание вдоль поперечной оси
- align-content - выравнивание флекс элементов, располагающихся в несколько строк

justify-content

Свойство управляет размещением элементов вдоль главной оси (горизонтальной, если flex-direction равно row, или вертикальной, если flex-direction равно column). Имеет значения:
- flex-start (значение по умолчанию): элементы выравниваются от начала основной оси;
- flex-end: элементы выравниваются по концу основной оси;
- center: элементы выравниваются по центру основной оси;
- space-between: (пространство между элементами), элементы равномерно распределяются по основной оси с равными промежутками между ним. При этом, первый и последний элементы, прилегают вплотную, соответственно, к началу и к концу оси. Так, если существует только два item, то они станут расположены по краям оси (контейнера);
- space-around: (пространство вокруг элементов, с несхлопывающими пространством между ними), элементы равномерно распределяются по основной оси (контейнеру), но с одинаковым пространством до и после каждого элемента. Таким образом, учитывая, что margin между элементами не схлопывается, он удваивается между элементами;
- space-evenly: (равномерное пространство между элементами), элементы равномерно распределяются по основной оси, при этом все промежутки равны между собой
