========================================== Корзина  =============================================

Блок корзины позволяет создавать обращение к ней для создания моментального перехода к оплате из любого блока. 
Например, существует товар "Крассовки t23452" со стоимостью 5400р. Для него, на любой странице сайта может быть создан элемент, при клике на который, производится переход к оплате за данный товар.

Элемент должен включать в себя id товара (#order:<productname>=<price>), поэтому для примера с кроссовками, ссылка кнопки станет включать в себя #order:Крассовки t23452=5400. Таким образом может создаваться отдельная страница с описанием особых товаров, которые могут включать иную от карточки товара стоимость, указывая её в специальном блоке-кнопке



========================================== Общее  =============================================

Существуют настройки как самого сайта, так и каждой отдельной страницы.
	- Настройки страницы относятся к конкретной странице сайта. Рекомендации:
		- Каждая страница имеет: заголовок(title), описание(description) и адрес этой страницы, а также бейджик(презентативное фото страницы), которое по умолчанию берется из первой картинки на странице
		- Для SEO оптимизации на вкладке "Facebook and SEO" можно задать специальные meta-данные, которые можно продублировать или прописать новые из соответствующих данных страницы, а также указать ключевые слова через запятую
		- При скачивании сайта архивом основной из проблем станет являться невозможность дальнейшего изменения (что логично), а также отключение оплачиваемых tilda шрифтов
	- Настройки всего сайта относятся ко всем страницам домена. Рекомендации:
		- Получение сертификатов безопасности HTTPS (настройка HTTPS) и включение редиректа с HTTP на HTTPS (редирект страниц)
		- Выполнение всех рекомендаций из категории SEO-оптимизация
		- Метрика предоставляет расширенную статистику сайта, вплоть до отслеживания действий пользователя на странице через личные кабинеты почты Яндекс или Гугл метрики. Соответственно, их подключение осуществляется через активирование данных опций
		- Фейсбук Pixel запоминает всех посетителей сайта, в дальнейшем, имя возможность ведения точечной рекламы по тем пользоваетлям, которые посетили сайт
		- "Google search console" и "Яндекс ВЕБМАСТЕР" применяются для индексирования поисковой системой существовующего сайта. Также предоставляют информацию о возможных проблемах индексирования и включают рекомендации в соответствующих кабинетах этих систем



========================================== Каталог товаров Tilda  =============================================

- Сохраненный каталог всех товаров сайта
	Каталог товаров представляет собой гугл таблицу для удобности взаимодействия, где собраны все товары всех категорий с описаниями, брендом, составом, габаритами и пр. Коррректная версия каталога с сайта, может быть получена путем его скачивания в формате csv. При этом для открытия потребуется использовать кодировку utf-8

- Для каталога на tilda предусмотрено два режима взаимодействия: обновление данных текущих товаров и загрузка новых товаров.
При этом, действует ограничение на размер загружаемого файла, он не должен превышать 3мб. Если файл больше 3мб - потребуется его разделение на несколько файлов, не превышающих 3мб.

- Обновление информации о существующих товарах в каталоге tilda
	Обновление информации о товарах осуществляется через загрузку в каталог csv файла, при этом, файл должен содержать только те товары, которые уже существуют на данный момент в каталоге tilda, в противном случае, процесс вызовет ошибку.

- Загрузка новых товаров в каталог tilda
	Загрузка новых товаров в католог осуществляется через csv файл. При этом, файл также может содержать товары, которые уже существуют на данный момент в каталоге tilda. Если товар существует в каталоге тильды (идентичен id товара) и если прогружаемые данные о нем отличаются от тех которые были ранее, то они перезапишутся новыми, т.е. осуществляя процесс обновления информации о товаре.
	
- Самым главным в каталоге товаров на тильде являются id товара - tilda UID каждого отдельного товара, в том числе каждый вид товара, как вариант от основного.
	Именно за счет данного значения, осуществляется нахождение и обновление товаров в каталоге
	Прогрузка файлов должна включать в себя id товара
	По неизвестным причинам, он не принимается системой и для каждого прогруженного товара, системой выдается новый
	Для актуализации id товаров, потребуется скачать каталог товаров tilda, взять из него id существующих товаров и скопировать в каталог сохраненных товаров. После чего, появится возможность взаимодействия с прогруженным в тильду товаром, например, его применение в кастомных карточках или обновление информации о нем, через прогрузку новых данных 



========================================== CSV файл =============================================

Если требуется скоповое внесение или обновление информации о товарах, то одним из вариантов является импортирование файла формате CSV.
Для открытия скачанного CSV документа в корректном виде:
	- потребуется изначально открыть программу Excel
	- в ней, через создание пустой таблицы, открыть скачанный документ через вкладку "Данные" -> "Получить данные" -> "Из файла CSV"
	- в открывшемся окне, потребуется выбрать кодировку "Unicode UTF-8", с необходимым разделителем, зачастую, это "точка с запятой"
	- Полученные данные, лучше всего скопировить и редактировать в Google sheets. И уже только подготовленный для отправки документ, скачивается в формате CSV. 
	
Редактирование данных в Google sheets:
	- Tilda UID — уникальный идентификатор товарной единицы в tilda (может потребоваться указать апостроф «'»)
		
	- SKU — идентификатор (может быть не уникальным на tilda) товарной единицы в системе учета пользователя (может потребоваться указать апостроф «'»)
		
	- Category — название раздела, куда будет отнесен товар, несколько разделов могут быть перечислены через точку с запятой
		
	- Title — название товара. Может быть сформирован функцией =СЦЕП: {Название товара} / {Название анализа}
		
	- Price — стоимость товара
		
	- Description — короткое описание, демонстрируется только на основной информации карточки товара, но не в ней самой. Может представлять собой =regexextract(<ссылка на Text>; "паттерн, например первой строки")
		
	- Text — полное описание товара. Может включать дополнительные разделы
		
	- Photo - ссылка на фото

	- Parent UID — обязательное поле, если присутствует принадлежность данного пункта другому товару. Заполняется из строки Tilda UID.
		
	- Brand — Бренд товара, системное свойство
		
	- Editions — Определенный вариант товара, который представляет собой конкретное свойство со своей ценой. 
	В колонке Editions могут быть перечислены необходимые свойства, где свойство товара имеет название, а через двоиточие указывается его значение.
	Свойств может быть множество, тогда они разделяются без пробелов точкой с запятой 
	Например: 
	Editions		Editions
	Упаковка:100гр.		Упаковка:100гр.
				Упаковка:200гр.
 	
		
	- Modifications - Дополнительная опция товара, зачастую допродажа или вариатив услуги, позволяет предложить дополнительный аксессуар
	В колонке Modifications могут быть перечислены до пяти дополнительных свойства, где свойство товара имеет название, а через двоиточие указывается его значение.	Это может быть обычная упаковка по умолчанию и подарочная + 500р, стандарный набор чехлов и дополнительный + 1200р и т.д.
	Разделителем между несколькими дополнительными опциями является альтернация: |
	Раделителем между несколькими разными вариантами дополнительных опция является точка с запятой: ;
	Примеры строки модификации:
	Подарочная упаковка +500руб. = +500
	Аксессуары:Нет;Дополнительный чехол=+3000|Сборка:Не нужна;Нужна=+1000
	Например:
	Modifications
	Упаковка:Стандартная упаковка;Подарочная упаковка +500руб.= +500				

	- Mark - Отметка на карточке (Например, SALE, NEW, -30%)

	- Charesteristics:<Пользовательская характеристика> - Дополнительный тип столбцов, который применяется для введения в товар дополнительной информации. Зачастую, применяются для введения значений, который станут применяться в фильтрах категории товара
	Например: 
	Characteristics:Страна произрастания	Characteristics:Форма листа
	Непал					Крупнолистовой чай

	- SEO title - Заголовок для SEO оптимизации
	- SEO descr - Описание для SEO оптимизации

	- Прочие столбцы могут быть включены для дополнительных операций, например предобработки для вынесения значений в столбец, который станет использоваться.
	Например, столбец "Text raw", который станет являться источником данных для столбца "Text", в котором будет определена формула, например, =regexrepace()



========================================== Табы в карточке товара =============================================

Табы для каждой карточки товара, чтобы отображать информацию о товаре в соответствующем ему разделе
	Вы можете создать несколько вкладок и выбрать, какую информацию в них показывать.

	Есть 4 типа вкладок на выбор:
	информация — уникальное описание для этого товара (текст, изображение, видео),
	шаблон — описание, которое можно добавить в один товар и потом выбрать его как шаблон в остальных товарах,
	текст — подробное описание товара из поля Текст,
	характеристики товара — информация о бренде, габаритах и других параметрах из раздела Характеристики.

	После настройки вкладок, включите их в настройках блока с товарами → Pop-up → «Табы в карточке товара» и укажите, как их нужно отображать — в виде вкладок или раскрывающихся карточек с текстом.



========================================== Перенос макета из Figma в Tilda =============================================
https://blog.tilda.cc/import-from-figma-to-zero-block

Должны соблюдаться правила:
	- Ширина фрейма — 1200px
	- Каждый блок расположен в отдельном фрейме
	- Чтобы импортировать группу изображением, назовите её image или svg
	- Для импорта кнопки объедините текст и шейп в группу и назовите её button. Ширина текста на кнопке должна совпадать с шириной шейпа кнопки. Если на шейпе есть эффект, который нельзя воспроизвести в Zero Block, он импортируется с изображением эффекта на фоне
